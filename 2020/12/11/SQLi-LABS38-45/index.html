堆叠注入

我本人不是很喜欢这种注入的方式。

局限性比较大：

1.后面的sql注入语句无论对错，是否执行成功，我们在前端界面是看不出来的。延时注入，报错注入就都不能用了。

2.mysqli_multi_query(connection,query)函数执行一个或多个针对数据库的查询。多个查询用分号进行分隔。必须要使用这个函数时才可以进行堆叠注入

参数 描述
connection 必需。规定要使用的 MySQL 连接。
query 必需。规定一个或多个查询，用分号进行分隔。

3.有些数据库本就不支持多个语句的查询,可能受到API或者数据库引擎不支持的限制，当然了权限不足也可以解释为什么攻击者无法修改数据或者调用一些程序。

我更推荐使用联合注入方法！实在没法子可以再使用堆叠注入！！

38-45这几关主要先找到注入点，然后再进行堆叠注入。每关只是注入点的不同！
