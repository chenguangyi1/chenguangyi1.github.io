php://filter的妙用

2021-08-15
  • 在CTF比赛中常常会见到
$data = $_GET['data'];
$filename = $_GET['filename'];
$content = "<?php ... exit();?>".$data;
file_put_contents($filename,$content);

因为存在exit()所以即使成功写入shell也执行不了。

  • 如何绕过这个死亡exit呢？

因为这里$filename可控，是可以控制协议的。所以可以使用php://filter协议的特性来除去exit。

  • php://filter介绍
参数 功能
read 读取
write 写入
resource 来源

read/write参数值可为

  • string.strip_tags： 将数据流中的所有html标签清除
  • string.toupper： 将数据流中的内容转换为大写
  • string.tolower： 将数据流中的内容转换为小写
  • convert.base64-encode / convert.base64-decode 将数据流中的内容转换为base64编码转换

常用的方式有

php://filter/read=convert.base64-encode/resource=/flag
读取文件
php://filter/string.strip_tags/resource=/etc/passwd 
在PHP<7.2可以引发内存错误
php://filter/write=string.rot13/resource=shell.php
利用rot13除去eixt，前提关闭了短标签形式
php://filter/write=convert.base64-decode/resource=shell.php

可以使用 php://filter/write=convert.base64-decode 来首先对其解码。在解码的过程中，字符<、?、;、>、空格等一共有7个字符不符合base64编码的字符范围将被忽略，所以最终被解码的字符仅有“phpexit”和我们传入的其他字符。

“phpexit”一共7个字符，因为base64算法解码时是4个byte一组，所以给他增加1个“a”一共8个字符。这样，"phpexita"被正常解码，而后面我们传入的webshell的base64内容也被正常解码。结果就是<?php exit; ?>没有了

这个<?php exit; ?>实际上是一个XML标签，既然是XML标签，我们就可以利用strip_tags函数去除它，而php://filter刚好是支持这个方法的。

我们最终的目的是写入一个webshell，而写入的webshell也是php代码，如果使用strip_tags同样会被去除。

php://filter允许使用多个过滤器，我们可以先将webshell用base64编码。在调用完成strip_tags后再进行base64-decode。“死亡exit”在第一步被去除，而webshell在第二步被还原。

php://filter/write=string.strip_tags|convert.base64-decode/resource=shell.php
