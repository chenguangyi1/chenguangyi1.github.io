正则表达式

2020-12-11

①例子：^[0-9]+abc$

  • ^表示匹配输入字符串的开始位置。yi
  • [0-9]匹配单个数字
  • [0-9]+匹配多个数字
  • +表示匹配一个或多个
  • abc$匹配字母abc并以abc结尾的字符串
  • $表示匹配输入字符串结束的位置

②^[a-z0-9_-]{3-15}$

规定只能用a-z，0-9，_ , - 。且长度在3-15之间。

var str = "a1d23abc";
var patt1 = /^[0-9a-z_]+/;      a1d23abc
var patt2 = /^[0-9a-z_]/;       a
var patt3 = /^[0-9a-z_]+abc$/;  a1d23abc
var patt4 = /abc$/;             abc
var patt5 = /b$/;               null

③ +，*，？的使用

  • runoo+b，+ 号代表前面的字符（即o）必须至少出现一次（1次或多次）。
  • runoo*b，* 号代表前面的字符（即o）可以不出现，也可以出现一次或者多次（0次、或1次、或多次）。
  • colou?r ,可以匹配 color 或者 colour，? 问号代表前面的字符（即u）最多只可以出现一次（0次、或1次）。

④[^]，/g, \s, \S,\w 的使用

  • /g 表示该表达式将用来在输入字符串中查找所有可能的匹配，返回的结果可以是多个。如果不加/g最多只会匹配一个
  • /[^abc】/g 匹配除括号里内容外的全部字符
  • /[…]/g 匹配里面的所有字符
  • /[a-z]/g 匹配小写a-z中的所有字母
  • /./g <=>/[^\n\r】/g 匹配除换行符（\n、\r）之外的任何单个字符。
  • /[\s\S]/g 匹配所有内容 （\s 是匹配所有空白符，包括换行，\S 非空白符，包括换行）
  • /[\w]/g <=> /\w/g <=> /[a-z0-9A-Z_]/g 匹配字母、数字、下划线。等价于 [A-Za-z0-9_]

注：是否有【】这个进行包裹！！

⑤特殊符号 $,(),*,+,.,[],?,,^,{},| 的使用

所谓特殊字符，就是一些有特殊含义的字符，如上面说的 runoo*b 中的 *，简单的说就是表示任何字符串的意思。如果要查找字符串中的 * 符号，则需要对 ***** 进行转义

即在其前加一个 \(反斜杆即转义符号):runo\*ob 匹配 runo*ob。
  • ^ 匹配输入字符串的开始位置
  • $ 匹配输入字符串的结尾位置
  • +匹配前子表达式一次或多次
  • *匹配前子表达式零，一次或多次
  • ？匹配前子表达式零或一次
  • . 匹配除换行符\n 外的任何字符
  • () 标记一个子表达式的开始和结束位置
  • [] 标志一个中括号表达式的开始
  • ^如果在[]外侧表示字符串的开始，在[^]里面表示除里面的字符外的其他所有字符。
  • |表示两项之间选择一个
  • {}表示限定字符表达式的开始，如上面的②例子
  • \ 标记下一个字符是特殊字符或文字

GBK和UTF-8文字编码的区别

2020-12-08

知识补充：

一个字节占8位即0000 0000。范围为-128~+127。-0表示为-128

注：查看Mysql数据库编码命令show variables like ‘character%’;

Variable_name            | Value                                                      |
+--------------------------+------------------------------------------------------------+
| character_set_client     | utf8                                                       |
| character_set_connection | utf8                                                       |
| character_set_database   | utf8                                                       |
| character_set_filesystem | binary                                                     |
| character_set_results    | utf8                                                       |
| character_set_server     | utf8                                                       |
| character_set_system     | utf8

①character_set_client 是指客户端发送过来的语句的编码方式;

②character_set_connection 是指server收到客户端的语句后，要转换成的编码方式；

③character_set_results 是指server执行语句后，返回给客户端的数据的编码。

数据传入
A1 客户端发送出语句(总是以utf8)------> A2 sever收到语句解码(按character_set_client指定编码)
                                                                    |
                                                                    v
A4 数据进入mysqld内部存储<--------- A3 sever判断是否需要转换编码(以character_set_connection 目标编码)

A1传进去的数据是以UTF-8进行编码的。A2接收到句子进行解码，如果set_client设置成GBK，服务器会以为传进来的数据是以GBK的方式进行编码，所以就以GBK编码方式进行句子的解码。然后再进行A3将解码后的句子再以set_connection指定的编码马上对句子进行编码。最后传入数据库里。

数据传出回显
B1 server返回结果(按character_set_results 指定编码) ----->B2客户端解码显示(总是以utf8)

B1 数据从数据库中取出显示，server会先按set_results指定的编码方式对数据进行编码，然后传到B2客户端进行UTF-8进行解码显示。

注：客户端发送的句子不一定都是以utf-8的编码方式进行编码，要看实际环境。

GBK编码

①是为了结合中国国情，因为汉字而衍生出的一种的编码方式。

②GBK的文字编码是用双字节来表示的，即不论中、英文字符均使用双字节来表示，但是为了区分中文，将其最高位置1.

③GBK码共收入汉字21003，符号883个，简体、繁体字融于一库。

④GBK是在国家标准GB2312基础上扩容后兼容GB2312的标准。

⑤一个GBK汉字占两个字节，取值范围是（编码位数）：第一个字节是（129-254），第二个字节（64-254）。

当设置GBK编码后，遇到连续两个字节，都符合GBK取值范围，会自动解析为一个汉字。

UTF-8编码

①UTF-8编码包含全世界所有国家需要用到的字符。

②UTF-8编码用于解决国际字符的一种多字节编码方式，对于英文字符则使用一字节来编码，中文汉字则使用三个字节来编码。

③UTF-8编码的文字可以正常显示在各国支持UTF-8编码字符集的浏览器上。

注：

UTF-8版本虽然具有良好的国际兼容性，但中文需要比GBK版本多占用50%的数据库存储空间，因此并非推荐使用，仅供对国际兼容性有特殊要求的用户使用。

总结：

①GBK就是在保存你的帖子的时候，一个汉字占用两个字节。外国人（使用UTF-8编码）看会出现乱码，GBK编码主要针对中国国情而出现的。

②UTF8就是在保存你的帖子的时候，一个汉字占用3个字节。但是外国人看的话不会乱码，此为西人为了解决多字节字符而形成之解决方案。

③对于英文较多的论坛使用UTF-8编码比GBK编码更节省储存空间。反之中文较多的论坛，如果大部分面向中国客户，使用GBK编码更合适。

④但是GBK仅针对中国国情而发明出来的，在国际上兼容性并不是很好，所以大多数的时候还是采用UTF-8编码。

TCP三次握手

2020-12-06

第一次握手：

发送端（客户端）发送连接请求将SYN=1,ACK=0,产生一个随机序列seq=x（Client 进入 SYN-SENT(同步已发送)状态）

第二次握手：

接收端（服务器端）接收到请求，确认的话会将（SYN=1,ACK=1,ack=x+1，产生随机序列seq=y ）发送给客户端请求连接（Server 进入 SYN-RCVD（同步收到）状态）

第三次握手：

客户端接收到服务器端请求的报文，确认ACK=1,ack=x+1后，将ACK=1，ack=y+1发送给服务器端。服务器端确认ack=y+1，ACK=1完之后，（Client 和 Server 都进入 ESTABLISHED（已建立连接）状态），通信通道建立完成，可以开始传输数据。

注：ack和ACK不同

ack代表的是头部的确认号Acknowledge number,是对上一个包的序号进行确认的号。ack=seq+1

ACK用于标志TCP包是否对上一个包进行确认操作，确认了置1。没确认则为0。

4.各种指令的使用

–current-user

当前数据库所使用的账户

–dbms=MySQL

指定数据库类型为MySQL

–file-read “路径\1.txt”

含义：读取目标服务器路径下的1.txt文件

注：使用前先查询当前用户是否拥有读取文件的权限

–file-write “路径\1.txt” –file-dest “D:\2.txt”

含义：将本地下的1.txt 文件的内容，写入到目标服务器下D盘下，并命名为2.txt。

注：使用该指令前需要先查询当前用户是否拥有写入文件的权限

–is-dba

当前用户是否拥有（root权限）

–level=1

From which level check for this test

1: Always (<100 requests)
2: Try a bit harder (100-200 requests)
3: Good number of requests (200-500 requests)
4: Extensive test (500-1000 requests)
5: You have plenty of time (>1000 requests)

默认使用level 1

–passwords

数据库密码

–risk=1

Likelihood(可能性) of a payload to damage the data integrity(完整性)

1: Low risk
2: Medium risk
3: High risk

–users

所有数据库用户
